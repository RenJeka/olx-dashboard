# OLX.ua — «API» запитів (HTML-ендпойнт пошуку)

> Документація того, **що саме ми викликаємо на OLX, як, з якими параметрами і що отримуємо**.
> OLX не має публічного API — ми використовуємо звичайну server-rendered HTML-сторінку пошуку.
> Якщо OLX щось поміняє — цей файл є базовою лінією «що було і працювало» (перевірено живими
> запитами **2026-06-10**).
>
> Повʼязане: [`architecture.md`](./architecture.md) §3–4, [`olx-monitor-spec.md`](./olx-monitor-spec.md) §4,
> код: `server/src/scraper/selectors.ts`, `server/src/scraper/olxFetcher.ts`.

---

## 1. Загальний принцип

Ми НЕ використовуємо жодного JSON API. Один тип запиту:

```
GET https://www.olx.ua/d/uk/list/q-<query-slug>/?<параметри>
```

Це **сторінка пошуку**, яку OLX рендерить на сервері (Next.js SSR). Картки оголошень уже
присутні в HTML — браузер/JS не потрібні. Парсимо HTML через cheerio.

**Чому не `api/v1/offers/`:** прихований JSON-ендпойнт підтверджений лише для olx.in, для
olx.ua — ні. Заборонено каноном (`CLAUDE.md`).

**Хто викликає:** `HtmlOlxFetcher.fetchSearch()` (`server/src/scraper/olxFetcher.ts`) —
один виклик на сторінку, до 3 сторінок на скан, із затримкою 1–2 с між сторінками.

## 2. Структура URL

### 2.1 Шлях

```
/d/uk/list/q-<query-slug>/
```

| Сегмент | Значення |
| --- | --- |
| `/d/` | "desktop" розділ сайту |
| `/uk/` | мова інтерфейсу (українська) |
| `/list/` | пошук по всіх категоріях |
| `q-<slug>` | пошуковий запит: lowercase, пробіли → `-`, потім URL-encode. `iphone 13` → `q-iphone-13` |

Пошук у категорії має інший шлях (`/d/uk/<category-path>/q-.../`) — ми це поки НЕ використовуємо
(`category_id` у схемі є, але в URL-білдер не підключений).

### 2.2 Query-параметри (ключ-значення)

Усі, що ми знаємо й використовуємо:

| Ключ | Значення | Використовуємо? | Призначення |
| --- | --- | --- | --- |
| `currency` | `UAH` | ✅ завжди | ціни в гривні |
| `search[order]` | `created_at:desc` | ✅ завжди | сортування: найновіші перші |
| `view` | `list` | ✅ завжди | формат видачі «список» (стабільніша розмітка) |
| `page` | `2`, `3`, … | ✅ (для стор. >1) | пагінація; перша сторінка — без параметра |
| `search[filter_float_<name>:from]` | число | ✅ (з `api_filters.ranges`) | нижня межа числового фільтра |
| `search[filter_float_<name>:to]` | число | ✅ (з `api_filters.ranges`) | верхня межа |
| `search[filter_enum_<name>][0]`, `[1]`… | рядок | ✅ (з `api_filters.enums`) | категорійний фільтр, по індексу на значення |
| `search[private_business]` | `private` | ✅ (з `api_filters.privateOnly`) | тільки приватні продавці |

Відомі імена фільтрів (`<name>`):

- `price` — єдиний універсальний float-фільтр, працює в будь-якій категорії:
  `search[filter_float_price:from]=8000&search[filter_float_price:to]=15000`
- Інші (`ram_gb`, `motorbike_engine_displacement` тощо) — **категорійно-залежні**; точне імʼя
  можна підгледіти в URL фільтрів на самому сайті OLX. Наш URL-білдер передає будь-яке імʼя
  з `searches.api_filters` без валідації.

Інші параметри сортування, які OLX підтримує (не використовуємо, але корисно знати):
`search[order]=filter_float_price:asc`, `filter_float_price:desc`, `relevance:desc`.

**Кодування:** дужки `[`/`]` та `:` в іменах параметрів лишаємо літеральними (так їх шле сам
сайт і так підтверджено робочим кодом), кодуються лише **значення** enum-фільтрів і slug.

### 2.3 Приклад повного URL (наш реальний запит)

```
https://www.olx.ua/d/uk/list/q-iphone-13/?currency=UAH&search[order]=created_at:desc&view=list&search[filter_float_price:from]=8000&search[filter_float_price:to]=15000
```

## 3. Заголовки запиту (ОБОВʼЯЗКОВІ)

Без них OLX може віддати капчу/іншу розмітку. Визначені в `selectors.ts` → `REQUEST_HEADERS`:

```http
User-Agent: Mozilla/5.0 (Windows NT 10.0; rv:91.0) Gecko/20100101 Firefox/91.0
Referer: <той самий URL, що й запит>
X-Client: DESKTOP
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
Accept-Language: uk-UA,uk;q=0.9,en;q=0.8
```

- `User-Agent` — реалістичний браузерний (з підтвердженого робочого коду lerdem/olx-parser).
- `Referer` = сам URL пошуку — імітація переходу всередині сайту.
- `X-Client: DESKTOP` — внутрішній заголовок OLX.

Авторизація/кукі — **не потрібні**.

## 4. Відповідь

### 4.1 High-level

- `200 OK`, `Content-Type: text/html` — повна Next.js сторінка (~1–2 МБ).
- Дані оголошень присутні **двічі**: у server-rendered HTML (ми парсимо це) і в
  `<script id="__NEXT_DATA__">` (JSON — наш fallback, див. §7).
- Стилі emotion (`css-*` класи) — **нестабільні**, тому парсимо ТІЛЬКИ по `data-cy` /
  `data-testid` атрибутах.

### 4.2 Спрощений приклад HTML (структура картки)

```html
<div data-cy="l-card" id="891834461">          <!-- id = olx_id, ключ дедуплікації -->
  <a href="/d/uk/obyavlenie/iphone-13-midnight-128gb-...-ID10ozqe.html?...">
    <img src="https://ireland.apollo.olxcdn.com/v1/files/.../image;s=216x152" />
  </a>
  <h4>Iphone 13 midnight 128GB neverlock • 100% АКБ</h4>   <!-- було h6, тепер h4! -->
  <p data-testid="ad-price">13 500 грн.</p>
  <p data-testid="location-date">Київ, Святошинський - Сьогодні о 06:19</p>
</div>
...
<!-- якщо нічого не знайдено: -->
<div data-cy="empty-state">...</div>
```

### 4.3 Селектори, які ми читаємо (`selectors.ts`)

| Поле | Селектор (відносно картки) | Примітки |
| --- | --- | --- |
| Картка | `[data-cy="l-card"]` | атрибут `id` картки = числовий `olx_id` |
| Назва | `h6, h4` | ⚠️ OLX мігрував `h6` → `h4` (зафіксовано 2026-06-10); тримаємо обидва |
| Ціна | `[data-testid="ad-price"]` | сирий текст: `"13 500 грн."`, `"550 $"`, `"Договірна"` |
| Лінк | `a[href]` (перший) | відносний → префіксуємо `https://www.olx.ua` |
| Фото | `img` (перший, лише `http*` src) | CDN: `ireland.apollo.olxcdn.com/v1/files/<id>/image;s=WxH` (розмір керується суфіксом `;s=`) |
| Локація/дата | `[data-testid="location-date"]` | формат `"<Місто[, Район]> - <дата>"`; дата: `"Сьогодні о 06:19"` або `"09 червня 2026 р."` |
| Порожня видача | `[data-cy="empty-state"]` | ознака «далі сторінок нема» |

Detail-сторінка оголошення (поки не використовується, Етапи 2+):
характеристики `[data-cy="ad-params"] li`, опис `[data-testid="ad_description"]`,
бізнес-продавець `[data-testid="trader-title"]`.

### 4.4 Що ми з цього робимо (нормалізація)

`normalizer.ts`:
- `"13 500 грн."` → `price=13500, currency='UAH'` (також розпізнає `$`→USD, `€`→EUR;
  без цифр → `price=null`);
- `"Київ, Святошинський - Сьогодні о 06:19"` → `city="Київ, Святошинський"`,
  `posted_at="Сьогодні о 06:19"` (дата зберігається сирим рядком);
- upsert у `listings` по `olx_id`.

### 4.5 Особливості видачі

- На сторінці ~40–50 карток; серед них є **promoted** (URL містить `search_reason=search|promoted`) —
  вони можуть повторюватися між сторінками, тому fetcher дедуплікує по `olxId` ще до БД.
- Картки без числового `id` (рекламні блоки) — пропускаються.
- Видача «з хвостом»: коли точних збігів мало, OLX доливає схожі товари (тому в пошуку
  «iphone 13» трапляються інші моделі) — фільтрація це наша справа (локальні фільтри, Етап 2).

## 5. Пагінація і ввічливість

- Сторінки: `?page=2`, `?page=3`. Ліміт у нас — **≤3 сторінки** на скан.
- Затримка **1–2 с** (рандомізована) між сторінками.
- Стоп-умови: `empty-state`, 0 карток, або сторінка не дала жодного нового `olxId`.
- Частота сканів — ручна/рідка; реалістичний UA. Це і є весь наш захист від rate-limit.

## 6. Помилки та нештатні відповіді

| Ситуація | Як проявляється | Наша реакція |
| --- | --- | --- |
| HTTP ≠ 200 | `res.ok === false` | виняток `OLX повернув HTTP <код>` → `scan_runs.error` |
| Порожня видача | є `[data-cy="empty-state"]` | нормальний стоп пагінації |
| Немає карток І немає empty-state | підозра на JS-only сторінку / капчу / зміну розмітки | виняток зі зразком перших 600 символів HTML + ознакою наявності `__NEXT_DATA__`; **НЕ** переходимо на браузер автоматично |
| Зміна селектора поля | поле порожнє у видачі (як було з `h6`→`h4`) | правити лише `selectors.ts` |

## 7. Fallback-стратегія (якщо HTML «зламається»)

Порядок (зафіксований у спеці §4.2, рішення приймає людина, не код):

1. **`__NEXT_DATA__`** — JSON у `<script id="__NEXT_DATA__">` тієї ж сторінки; містить ті самі
   оголошення структуровано. Перший кандидат, бо не потребує браузера.
2. Playwright з **видимим** Chromium (headless блокується) — крайній випадок, лише локально.

Архітектурно зміна стратегії = нова реалізація інтерфейсу `OlxFetcher` (`server/src/types.ts`),
решта системи не чіпається.

## 8. Чекліст «OLX щось поміняв»

1. Скан повернув 0 / помилку → подивитись `scan_runs.error` (там зразок HTML).
2. Якщо картки є, але поля порожні → звірити селектори §4.3 із живим HTML, оновити `selectors.ts`.
3. Якщо карток нема взагалі → перевірити `__NEXT_DATA__` (§7.1).
4. Після фіксу — оновити ЦЕЙ файл (дата + що змінилось) і `CLAUDE.md`, якщо змінився канон.
