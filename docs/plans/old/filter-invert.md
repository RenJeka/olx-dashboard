# Інвертований режим локальних фільтрів

## Контекст

Локальні фільтри (ціна, міста, продавці, плюси, мінуси) працюють як білий список.
Додаємо перемикач «Інвертувати» навпроти кожної секції в Drawer фільтрів. У інвертованому
режимі збіги **приховуються** (чорний список) — замість того, щоб показуватись.

Кожна група незалежна: один фільтр може бути прямим, інший — інвертованим.
Комбінування між групами — AND (як і раніше).

Зворотна сумісність: `invert` відсутній / `false` = поточна поведінка (прямий режим).

## Test-cases

1. **Build:** `npm run build` без TS-помилок.
2. **Зворотна сумісність:** пошуки без `invert` фільтрують як і раніше.
3. **Drawer UI:** у кожної секції праворуч є Switch «Інвертувати»; текст під назвою
   змінюється при перемиканні.
4. **Інверсія міста:** обрати місто + invert → у «ВІДФІЛЬТРОВАНІ» ці оголошення зникають.
5. **Інверсія мінуса:** обрати мінус + invert → приховуються оголошення з цим мінусом;
   незаналізовані — показуються.
6. **Інверсія ціни:** діапазон + invert → приховуються оголошення в межах діапазону;
   ціна IS NULL — не приховуються.
7. **AND:** прямий фільтр міста + інвертований мінус — приховується, якщо будь-яке правило
   спрацьовує.
8. **Скан:** upsert коректно перераховує `filtered_out` з інвертованими фільтрами.
